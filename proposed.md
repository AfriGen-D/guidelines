## Candidate guidelines:

This file is for collecting suggested guidelines. It might make sense to discuss individual topics as github issues

### Commit messages

Option 1. follow the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/#summary) standard.

  - `fix:`
  - `feat:`
  - `perf:`
  - `style:`
  - `docs:`
  - `refactor:`
  - `BREAKING CHANGE:`
  - others `ci:`, `build:`, `test:`, `chore:`

### Versioning

Option 1. https://semver.org/
  
>  Given a version number MAJOR.MINOR.PATCH, increment the:
>  
>      MAJOR version when you make incompatible API changes
>      MINOR version when you add functionality in a backward compatible manner
>      PATCH version when you make backward compatible bug fixes
>  
>  Additional labels for pre-release and build metadata are available as extensions to the MAJOR.MINOR.PATCH format.
